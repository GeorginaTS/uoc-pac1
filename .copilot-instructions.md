# Copilot Instructions for HTML + CSS Practice Project

## Project Overview
This is a Frontend practice project for UOC (Universitat Oberta de Catalunya) focused on creating a web page using pure HTML5 and CSS3. The project emphasizes semantic HTML, responsive design, and web accessibility standards.

## Project Structure Guidelines
```
practica1/
├── index.html          # Main HTML file
├── css/               # Stylesheets directory
│   ├── styles.css     # Main stylesheet
│   └── responsive.css # Responsive styles (if needed)
├── img/               # Images directory
├── js/                # JavaScript directory (if needed)
├── assets/            # Additional resources
└── README.md          # Project documentation
```

## Development Standards

### HTML Guidelines
- Use semantic HTML5 elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`)
- Include proper meta tags for viewport, charset, and description
- Ensure all images have alt attributes for accessibility
- Use heading hierarchy properly (h1 > h2 > h3, etc.)
- Include proper form labels and validation
- Validate HTML with W3C validator

### CSS Guidelines
- Use CSS3 features and modern properties
- Follow BEM methodology for class naming: `.block__element--modifier`
- Organize CSS with logical sections and comments
- Use CSS custom properties (variables) for consistency
- Implement mobile-first responsive design
- Use Flexbox and/or CSS Grid for layouts
- Avoid using CSS frameworks - pure CSS only
- Validate CSS with W3C validator

### Responsive Design Requirements
- Mobile-first approach (min-width media queries)
- Breakpoints: 
  - Mobile: up to 767px
  - Tablet: 768px - 1023px
  - Desktop: 1024px and above
- Flexible images and media
- Touch-friendly interface elements (minimum 44px touch targets)

### Accessibility Standards
- Follow WCAG 2.1 AA guidelines
- Ensure proper color contrast ratios (4.5:1 for normal text)
- Include focus states for interactive elements
- Use ARIA attributes when necessary
- Test with screen readers
- Ensure keyboard navigation works properly

### Code Quality Standards
- Use consistent indentation (2 or 4 spaces)
- Add meaningful comments to explain complex sections
- Keep code clean and readable
- Use semantic and descriptive naming
- Avoid inline styles - separate HTML and CSS
- Optimize images for web performance

### File Organization
- Keep HTML files in root directory
- Place all CSS files in `/css/` directory
- Store images in `/img/` directory with descriptive names
- Use lowercase file names with hyphens for separation
- Organize CSS properties in logical order: positioning, box model, typography, visual, misc

### Browser Compatibility
- Support modern browsers: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- Test cross-browser compatibility
- Use vendor prefixes when necessary
- Provide fallbacks for newer CSS features

### Performance Guidelines
- Optimize images (use appropriate formats: WebP, JPEG, PNG)
- Minimize CSS and HTML when ready for production
- Use efficient CSS selectors
- Avoid deep nesting in CSS
- Keep file sizes reasonable

### Documentation Standards
- Update README.md in Catalan for public documentation
- Comment complex CSS rules and HTML structures
- Include inline documentation for JavaScript (if used)
- Document any external resources or fonts used

### Version Control
- Use semantic commit messages in English
- Commit frequently with meaningful changes
- Follow conventional commits format: `type(scope): description`
- Types: feat, fix, docs, style, refactor, test, chore

## Specific Instructions for Copilot

### When writing HTML:
- Always start with proper DOCTYPE and HTML5 structure
- Include viewport meta tag for responsive design
- Use semantic elements appropriately
- Add alt text to all images
- Include proper form validation attributes

### When writing CSS:
- Start with CSS reset or normalize if needed
- Use mobile-first media queries
- Group related styles together with comments
- Use consistent naming conventions (BEM)
- Include hover and focus states for interactive elements

### When making changes:
- Always consider accessibility implications
- Test responsive behavior at different screen sizes
- Validate HTML and CSS after changes
- Keep the project structure organized
- Update documentation if structure changes

### Language Preferences:
- Code comments in English
- README.md and user-facing documentation in Catalan
- Variable and class names in English
- Commit messages in English

## Quality Checklist
